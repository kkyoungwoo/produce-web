# current-task.md

## 작업명
mp4Creater Step 흐름 안정화 + Scene Studio 이미지/영상 생성 UX 정리

## 현재 우선순위
1. Step 1~6 이동 시 프로젝트 문맥(`projectId`)을 유지하고 입력값 유실을 막는다.
2. Step 3 프롬프트 편집은 기본 프롬프트 원본을 보호하고, 프로젝트 복사본만 수정되도록 고정한다.
3. Step 6 진입 성능을 local-first로 유지하고 자동 생성 없이 사용자 액션 기반 생성으로 통일한다.
4. Scene 단위 생성/재생성/미리보기 경로를 분리해 실패 시에도 다음 작업으로 진행 가능하게 유지한다.

## 최신 반영 사항 (2026-03-20)

### Step 3 프롬프트 모달
- `프롬프트 보기` 모달에서 `이 프롬프트 수정` 클릭 시 즉시 저장되도록 수정.
- 모달 textarea 값과 템플릿 상태 동기화 키를 분리해, 저장 후에도 변경 내용이 모달에 바로 반영되도록 수정.
- 기본 프롬프트(`builtIn`)는 직접 수정하지 않고, 프로젝트 전용 복사본으로 분기한 뒤 편집하도록 유지.

### Step 6 Scene Studio 진입/생성
- Step 6 진입 시 전체 이미지/영상 자동 생성을 시작하지 않음.
- Step 5 -> Step 6 이동 시 프로젝트 초기 assets를 "빈 썸네일" 상태로 저장.
- Scene 카드에서 이미지가 없으면 `빈 썸네일` + `이미지 생성` 버튼 표시.
- 상단/결과영역 모두 `전체 이미지 생성` 버튼으로 이미지 일괄 생성 가능.
- 영상 생성은 항상 `현재 씬 이미지 + 비주얼/모션 프롬프트` 기준으로 씬별/일괄 생성.

### 미리보기
- 결과 미리보기에서 요청 시 씬들을 합친 합본 영상 프리뷰를 생성해 재생.
- 합본 프리뷰는 클릭 시점에 준비되며, Step 6 진입 시 자동 렌더하지 않음.

## 금지/주의 사항
- `db-cleanup`, `workbench`는 명시 요청 없으면 수정 금지.
- 기본 프롬프트 원본 텍스트는 수정 금지(프로젝트 복사본만 변경).
- API 미연결 상태에서도 샘플 fallback 흐름은 유지.
- `types.ts` 변경 시 InputSection/SceneStudioPage/서비스 호출부를 함께 점검.

## 검증 명령
- `npx tsc --noEmit --pretty false`
- `npm run build`
- `npm run lint` (현재 repo 전역 UTF-8 BOM 검증 실패 항목 존재)

## 수동 시나리오 체크
1. Step 3에서 `프롬프트 보기` -> 모달 수정 -> 저장 후 모달/카드 반영 확인
2. Step 5에서 `영상 생성` 클릭 후 Step 6 진입 시 빈 썸네일 상태 확인
3. Step 6에서 씬별 `이미지 생성` 버튼 동작 확인
4. Step 6에서 `전체 이미지 생성` 후 `모든 씬 영상 생성` 동작 확인
5. `전체 미리보기` 클릭 시 합본 영상 생성/재생 확인
