# 에이전트 공통 안내서

이 문서는 이 프로젝트를 수정하는 AI 작업자를 위한 공통 규칙이다.

## 1) 작업 원칙
- 항상 **최신 반영본 기준**으로 이어서 수정한다.
- 수정 범위는 **요청 범위로 최소화**한다.
- 정상 동작 중인 기존 기능은 건드리지 않는다.
- 관련 없는 리팩토링, 폴더 정리, 파일명 변경, 구조 변경, 포맷팅 정리 금지.
- 새 파일은 꼭 필요할 때만 만든다.
- 요청하지 않은 UI 변경 금지.
- 문제가 있어도 전체 재작성 대신 **국소 패치**를 우선한다.

## 2) MD 확인 규칙
사용자가 작업을 요청하면 먼저 아래 순서로 관련 문서를 찾는다.

1. **수정 대상 파일과 같은 폴더의 md**
2. **상위 폴더의 md**
3. 프로젝트 루트의 md
4. 요청과 직접 관련된 특정 가이드 md

규칙:
- 모든 md를 읽으려 하지 말고 **이번 요청과 직접 관련된 md만** 본다.
- 여러 md가 충돌하면 **더 가까운 경로의 문서**를 우선한다.
- 그래도 충돌하면 **사용자 최신 지시**를 최우선으로 따른다.

## 3) 한국어 인코딩 규칙
- 모든 텍스트 파일은 **UTF-8 기준**으로 유지한다.
- 한글이 깨질 수 있는 저장 방식(BOM 변경, 잘못된 재인코딩, 불필요한 변환) 금지.
- 문자열 수정 시 기존 한글 문구를 임의로 재작성하지 않는다.
- 인코딩 문제가 의심되면 **해당 파일만 최소 수정**한다.
- CSV, JSON, TS, TSX, MD 저장 시 한글이 그대로 보이는지 우선 확인한다.

## 4) 토큰 최소화 규칙
- 답변은 짧고 명확하게 쓴다.
- 장황한 설명, 불필요한 배경 설명, 긴 리팩토링 제안 금지.
- 필요한 파일만 식별하고 그 파일만 수정한다.
- 코드도 필요한 부분만 바꾸고 주변 코드는 유지한다.
- 요약은 파일별 1~2줄이면 충분하다.

## 5) 수정 절차
1. 요청 범위를 한 줄로 정리한다.
2. 관련 파일만 찾는다.
3. 관련 md가 있으면 필요한 부분만 확인한다.
4. 최소 패치로 수정한다.
5. 기존 동작 영향 여부만 짧게 점검한다.
6. **수정된 파일만** 상대경로 유지해서 전달한다.

## 6) 코딩 규칙
- 타입 오류는 **요청 범위 안에서만** 해결한다.
- 기존 함수명, 파일명, import 구조는 가능하면 유지한다.
- 동작 수정이 아닌 스타일 수정도 요청 범위 안에서만 한다.
- 저장/복원/복사/가져오기 로직은 서로 연결되므로 한쪽만 바꾸고 깨지지 않게 본다.

## 7) 우선순위
1. 사용자 최신 요청
2. 수정 대상 근처의 md
3. 상위 폴더 md
4. 루트 md
5. 기존 코드 관례

## 8) 금지 사항
- 요청 없는 대규모 정리
- unrelated 파일 수정
- 동작과 무관한 콘솔/주석 정리
- 임의의 구조 개선
- 과거 원본 기준으로 되돌려 수정

## 9) 한 줄 기준
항상 **최신 수정본 기준 / 관련 md 우선 확인 / UTF-8 한글 보존 / 최소 토큰 / 최소 패치 / 수정 파일만 전달**로 작업한다.
