# IMPLEMENTATION_AGENT_RULES.md

이 문서는 구현 에이전트의 **실행 기준**입니다.  
장문 분석보다 **최소 수정 + 안정성 유지 + hook 기반 검증**을 우선합니다.

## 1) 작업 분류
- 요청 시작 시 아래 4개 중 하나로 분류: `BUILD` / `FIX` / `CLEAN` / `REVIEW`
- 분류 후 5줄 이내 짧은 계획을 먼저 제시

## 2) 토큰 절약 원칙
- 전체 프로젝트 재탐색 금지, 요청 관련 파일만 확인
- 한 번에 1~3개 파일만 수정
- 성공 로그 장문 출력 금지, 실패 원인만 짧게 보고
- 설명보다 코드 결과 우선

## 3) bash hook 우선 검사 (대화로 반복 설명 금지)
아래 검사는 hook으로 처리하고, 실패 시에만 최소 수정:
- API 키/시크릿/비밀번호 하드코딩 검사
- `NEXT_PUBLIC_` 민감정보 노출 패턴 검사
- 위험한 SQL/migration 패턴 검사
- TypeScript/JSON 구문 검사
- 파일 비대화 검사
- Next.js hydration 위험 패턴 검사
- 잘못된 import / 없는 패키지 import 검사

## 4) mp4Creater 안정성 가드레일
- `scene-studio -> step-6` 이동 구조를 깨지 않음
- `projectId` 전달/유지 흐름을 끊지 않음
- 스텝 기반 제작 흐름(`step-1`~`step-6`)의 경로/상태 일관성 유지
- 한국어 입력 유지, 번역/변환은 전송 직전 규칙을 존중

## 5) AI 미연결/실패 시 동작 원칙
- API 키 없음/AI 호출 실패 시 에러 화면 대신 sample/fallback 유지
- 화면이 빈 상태로 멈추지 않게 기본 데이터/안내 문구 제공
- 기능이 부분 실패해도 다음 단계 진행 가능 상태 유지

## 6) 프론트/Next.js 안전 원칙
- 전체 페이지 블로킹 로딩보다 카드/패널 단위 부분 로딩 우선
- hydration failed, import 오류, client/server 경계 오류 회귀 금지
- 서버 컴포넌트에서 `window/document/localStorage/sessionStorage` 직접 사용 금지
- `use client` 필요 파일은 명시, `ssr: false`는 최소 사용
- 없는 패키지 import 금지

## 7) 수정 스타일
- 최소 수정 우선, 기존 네이밍/구조 존중
- 수정 범위 밖 대규모 리팩토링 금지
- 큰 파일은 점진 분리하되 기존 흐름 호환 우선

## 8) 응답 포맷
항상 아래 순서로 짧게 보고:
1. 작업 분류
2. 수정 계획(최대 5줄)
3. 수정 파일(1~3개)
4. 핵심 변경점
5. 실패한 hook/build 원인과 해결(실패가 있을 때만)
6. 성공 로그 장문 출력 금지
